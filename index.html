Введение

Прикладное ПО (или приложение) — «вспомогательная программа», предназначенная для выполнения определенных задач и рассчитанная на непосредственное взаимодействие с пользователем. В большинстве ОС прикладные программы не могут обращаться к ресурсам компьютера напрямую, а взаимодействуют с оборудованием и другими программами посредством операционной системы. К прикладному ПО относятся компьютерные программы, написанные для пользователей или самими пользователями для задания компьютеру конкретной работы: программы обработки заказов, создания списков рассылки, текстовые и графические редакторы, веб-браузеры, медиаплееры, компьютерные игры, программы-клиенты для электронной почты и блогов и др. Прикладное ПО используется, в частности, для управления информацией, базами данных, сетью и безопасностью. Также существует образовательное и имитационное ПО: первое способствует изучению того или иного материала и имеет четкие требования по тестированию знаний пользователя и отслеживанию его прогресса, второе используется для симуляции физических или абстрактных систем в целях научных исследований, обучения или развлечения.

Данная программа выводит конкретные данные из базы таксистов. Реализует следующий функционал:

  • список водителей и их авто
  • штрафы, похвалу
  • вызовы за последние сутки
  • оплата за последние сутки
  • то же самое для одного водителя по фамилии
  • всех водителей, имеющих штрафы
  • всех водителей, имеющих похвалы
  • всех, кто брал вызовы в некотором районе
  • информацию о водителе по его фамилии
  • информацию об авто по его номеру

1.2. Генерация водителей

Функция генерации водителя

Не принимает аргументов, рандомно генерирует данные, возвращает словарь данных. Cоздан список имён и программа рандомит имена водителей и их пол.

Функция генерации вызова

Не принимает аргументов, рандомно генерирует данные, возвращает словарь данных. Рандомно генерируется адрес из списка улиц, дом, длинна поездки, цена.

Функция генерации авто

Не принимает аргументов, рандомно генерирует данные, возвращает словарь данных. Рандомно генерируется номер авто по шаблону номеров РФ, марка и цвет выбираются из заданных списков.

1.4. Заполнение базы данных

Скрипт использует функции из прошлых файлов и генерирует данные, при помощи функций служебного класса добавляет их в базу данных.

В цикле генерирует данные и распаковывает по переменным. Передает переменные в функции, которые в свою очередь передадут их в базу данных.

1.5. Форматирование данных

Функция форматирования данных о водителях

Принимает фамилию или похвалы или штрафы. Делает соответствующий запрос в базу данных. В цикле проходится по результату запроса, превращая в данные в строку. Возвращает массив строк.

Функция форматирования данных о зоне вызова/прибытия

Принимает зону вызова/прибытия. Делает соответствующий запрос в базу данных. В цикле проходится по результату запроса, превращая в данные в строку. Возвращает массив строк.

Функция форматирования данных об авто

Принимает id авто или номер. Делает соответствующий запрос в базу данных. В цикле проходится по результату запроса, превращая в данные в строку. Возвращает массив строк.

Функция форматирования данных об авто по номеру

Принимает номер автор. Делает соответствующий запрос в базу данных. В цикле проходится по результату запроса, превращая в данные в строку. Возвращает массив строк.

2. Программа

2.1. Главный файл программы

Главный файл проекта, в котором декорируются функции из модуля форматированных данных. Файл необходим для создания рабочего окна программы и возвращения значений из exposed функций.

  • @eel.expose - Данный декоратор оборачивает Python функцию и получает ее возвращаемое значение, чтобы передать его в одноименную JavaScript функцию. Таким образом eel библиотека является связующим звеном между локальным сервером (компьютером, на котором запущено приложение) и браузером (который принимает значение посредством POST запросов от eel библиотеки).
  • get_drivers - Возвращает водителей, в зависимости от аргументов.
  • get_by_fine - Возвращает всех водителей со штрафами.
  • get_by_commendation - Возвращает всех водителей с пахвалами.
  • get_by_area - Возвращает водителей по вызову в определенном месте.
  • get_auto_by_num - Возвращает авто по номеру.
  • eel.start - Создает отдельный процесс, со своим портом и каналом для передачи данных. Также запускает браузер и выбранную html страницу.

2.2. Описание интерфейса

Интерфейс представляет из себя сетку состояющюю из: 6 кнопок, которые предназначены для переключения функционала программы. Поля ввода данных. Кнопки, которая даст сигнал скрипту к забросу данных от локального сервера, по выбранному условию.

2.3. Интерактивный скрипт

Файл который отвечает за корректное отображение данных которые мы получаем из eel функций. Также в нем описаны функции используемые в eventListener`ах

Функции слушающие события

В зависимости от функции отключают/включают поле ввода. Каждая функция содержит одинаковый код, с отличием в одноименную функцию из Python файла, которая возвращает им форматированные данные, которые JavaScript функция передает в HTML документ и тем самым формирует внешний вид программы и ее интерактив.
